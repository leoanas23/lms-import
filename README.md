# GO Import — LMS to GrowthWheel Online (web app)

Web version of the `client-import-lms-to-go` Claude skill. Upload raw TalentLMS
per-course .xlsx files, review matches, verify companies, and download the four
GO import files — with optional Drive archival and AI-assisted review.

Built from the validated skill spec (June 2026): `import-process-steps.md`,
`trainings-import-spec.md`, SKILL.md field overrides. The skill reference files
remain the source of truth — when rules change there, update `config/*.json`
and `lib/engine/` here.

## What it produces per session
1. `*_client-data_new_import.csv` (105 cols, template-exact)
2. `*_client-data_existing_import.csv`
3. `*_trainings_import.csv` (51 cols, UUIDv7 UIDs, `MMM D |` titles)
4. `*_training-participants_import.csv` (8 cols incl. blank F/H)

Plus a session log (AI-drafted when a key is configured) and Drive archival.

## Local development
```bash
npm install
cp .env.example .env.local   # set APP_PASSWORD at minimum
npm run dev                  # http://localhost:3000
npm test                     # engine test suite (46 checks)
```
Without Blob configured, sessions persist to /tmp (fine locally; on Vercel
you need Blob — see below).

## Deploy to Vercel
1. Push this folder to a new GitHub repo:
   ```bash
   git init && git add -A && git commit -m "GO import app v1"
   git remote add origin https://github.com/leoanas23/lms-import-app.git
   git push -u origin main
   ```
2. In Vercel: **Add New → Project → Import** the repo. Framework auto-detects Next.js. Deploy.
3. **Storage → Create → Blob** and connect it to the project
   (sets `BLOB_READ_WRITE_TOKEN` automatically). Required in production —
   serverless /tmp does not persist between requests.
4. **Settings → Environment Variables**:
   - `APP_PASSWORD` — team password (required)
   - `ANTHROPIC_API_KEY` — optional; enables company-flagging + AI session logs
   - `GOOGLE_SERVICE_ACCOUNT_JSON`, `DRIVE_FOLDER_OUTPUT`,
     `DRIVE_FOLDER_ARCHIVE`, `DRIVE_FOLDER_LOGS` — optional; enables Drive archival
5. Redeploy after setting env vars.

## Google Drive archival setup (optional)
1. Google Cloud Console → create project → enable **Google Drive API**.
2. Create a **service account**, generate a JSON key.
3. Share these Drive folders with the service account's email
   (Editor): OUTPUT - Files to import to GO, ARCHIVE - Files imported,
   Dataimport/import-logs.
4. Paste the JSON (single line) into `GOOGLE_SERVICE_ACCOUNT_JSON` and the
   three folder IDs into the DRIVE_FOLDER_* vars.

Without these vars the app still works fully — outputs are download-only.

## Where the rules live
| Rules | File |
|---|---|
| Output column orders (exact, from templates) | `config/*-columns.json` |
| Per-center defaults, territories, aliases | `config/customers.json` |
| Trainings fallbacks vs forced overrides | `config/trainings-defaults.json` |
| MD county canonical list | `config/counties-md.json` |
| Field mapping + SBA defaults + transformations | `lib/engine/clientRow.ts` |
| Duplicate matching (email → fullname, ambiguous) | `lib/engine/match.ts` |
| Company placeholder / legal entity | `lib/engine/company.ts` |
| Title rule, trainings + participants generation | `lib/engine/trainings.ts` |
| Validation checklist | `lib/engine/pipeline.ts` |

## Adding a new customer/center
Add a center entry in `config/customers.json` (advisor email, city/state/zip,
territory list, alias) — no code changes needed.
